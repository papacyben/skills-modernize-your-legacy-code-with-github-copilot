# COBOL 應用程式測試計劃

此測試計劃覆蓋目前 `src/cobol` 應用程式中的所有業務邏輯。它可供利害關係人驗證實現，也可作為將來在 Node.js 應用程式中建立單元測試和整合測試的參考。

| 測試用例ID | 測試用例描述 | 前提條件 | 測試步驟 | 預期結果 | 實際結果 | 狀態（通過/失敗） | 評論 |
|---|---|---|---|---|---|---|---|
| TC-001 | 檢視初始帳戶餘額 | 應用程式已啟動；帳戶初始餘額為 1000.00 | 1. 啟動程式
2. 選擇選項 `1` | 顯示當前餘額為 `1000.00` |  |  |  |
| TC-002 | 存款金額並確認餘額更新 | 應用程式已啟動；帳戶初始餘額為 1000.00 | 1. 啟動程式
2. 選擇選項 `2`
3. 輸入 `200.00` | 顯示存款後新餘額為 `1200.00` |  |  |  |
| TC-003 | 提款金額且餘額足夠 | 應用程式已啟動；帳戶餘額為 1000.00 或已更新後足夠 | 1. 啟動程式
2. 選擇選項 `3`
3. 輸入 `200.00` | 提款成功，顯示新餘額為 `800.00` |  |  |  |
| TC-004 | 提款金額但餘額不足 | 應用程式已啟動；帳戶餘額小於提款金額 | 1. 啟動程式
2. 選擇選項 `3`
3. 輸入如 `2000.00` | 顯示 `Insufficient funds for this debit.`，餘額不改變 |  |  |  |
| TC-005 | 選單輸入無效選項時顯示錯誤 | 應用程式已啟動 | 1. 啟動程式
2. 輸入 `5` 或其他非 `1-4` 選項 | 顯示 `Invalid choice, please select 1-4.`，程式繼續運行 |  |  |  |
| TC-006 | 正常退出程式 | 應用程式已啟動 | 1. 啟動程式
2. 選擇選項 `4` | 顯示 `Exiting the program. Goodbye!` 並結束程式 |  |  |  |

## 備註

- 目前 COBOL 應用程式僅管理單一帳戶，沒有學生識別或多帳戶管理邏輯。
- 將來將此測試計劃轉換為 Node.js 應用程式時，應將以下業務規則納入測試：
  - 初始帳戶餘額為 `1000.00`
  - 存款可更新餘額
  - 提款前須檢查帳戶是否足夠
  - 不足餘額時應提供明確錯誤訊息
  - 選單輸入僅接受 `1-4`
